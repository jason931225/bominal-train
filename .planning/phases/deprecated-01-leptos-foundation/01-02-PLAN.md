  <name>Task 2: Create minimal App with hydrate entry point and verify dual compilation</name>
  <files>crates/bominal-app/src/lib.rs</files>
  <action>
    **Step 1: Ensure wasm32-unknown-unknown target is installed**

    Run `rustup target add wasm32-unknown-unknown` to ensure the WASM compilation target is available.

    **Step 2: Rewrite lib.rs with both SSR and hydrate entry points**

    Update `crates/bominal-app/src/lib.rs` to:

    ```rust
    pub mod i18n;

    use leptos::prelude::*;

    /// Root application component.
    ///
    /// Phase 1: Minimal skeleton that proves SSR and hydration work.
    /// Phase 3 will add the full router, layout, and auth guard.
    #[component]
    pub fn App() -> impl IntoView {
        view! {
            <p>"Bominal - Leptos SSR"</p>
        }
    }

    /// Client-side hydration entry point.
    /// Called by the WASM bundle when loaded in the browser.
    #[cfg(feature = "hydrate")]
    #[wasm_bindgen::prelude::wasm_bindgen]
    pub fn hydrate() {
        leptos::mount::hydrate_islands();
    }
    ```

    Key points:
    - The `hydrate()` function uses `hydrate_islands()` (not `hydrate_body` or `mount_to_body`) because we are using islands architecture. Only `#[island]` components get hydrated.
    - The `#[cfg(feature = "hydrate")]` gate ensures this function only compiles for the WASM target.
    - The `pub mod i18n;` keeps the stub from plan 01-01.
    - No `#[wasm_bindgen(start)]` — the `hydrate()` function is called explicitly by the cargo-leptos generated script tag.

    **Step 3: Verify both compilation targets**

    Run both checks:
    1. `cargo check -p bominal-app --features ssr` — verifies SSR compilation
    2. `cargo check -p bominal-app --target wasm32-unknown-unknown --features hydrate` — verifies WASM compilation

    Both must succeed with no errors.

    If the WASM check fails due to web-sys features or other issues, fix the Cargo.toml accordingly. Common issues:
    - Missing `getrandom/js` feature for wasm32
    - web-sys features that don't exist in the version — remove them and add back as needed
    - `leptos_axum` or `reqwest` leaking into WASM build — ensure they are behind `optional = true` and only in the `ssr` feature

    **Step 4: Verify workspace still compiles**

    Run `cargo check --workspace` to ensure the bominal-app changes don't break other crates.

    Per FND-05: Dual compilation verified.

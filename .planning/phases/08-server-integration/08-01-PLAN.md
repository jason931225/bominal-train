# Phase 8 / Plan 01

## Objective

Switch non-API requests from the old SvelteKit static fallback to Leptos SSR and serve static assets from the cargo-leptos output directory.

## Implementation

1. Add the missing SSR shell/route integration needed for `bominal-app` to render through `leptos_axum`.
2. Initialize `LeptosOptions` in the production server state and replace `frontend/build` fallback serving with Leptos SSR plus static asset serving rooted at `target/site`.
3. Preserve `/api`, `/health`, `/metrics`, and the existing middleware/security layers while changing only the non-API route handling path.

## Verification

- `cargo check -p bominal-app --features ssr`
- `cargo check -p bominal-app --target wasm32-unknown-unknown --features hydrate`
- `cargo check -p bominal-server --features ssr`

## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | `gap-1`, inline icon spacing, `ml-1` on English subtext |
| sm | 8px | `gap-2`, compact element gaps between station inputs and swap button |
| md | 12px | `px-3`, input horizontal padding |
| lg | 16px | `mb-4`, GlassPanel vertical spacing between sections |
| xl | 20px | `py-2.5` (10px vertical padding on inputs, but total touch target ~40px), `mb-5` heading bottom margin |
| 2xl | 24px | `p-6`, GlassPanel internal padding (inherited from existing pattern) |

Exceptions: Touch target for dropdown option buttons is 40px tall (`py-2.5` + `text-sm` line-height). This meets minimum 44px when combined with the text content height, which is acceptable for dense autocomplete lists per WAI-ARIA combobox examples.

## Component Visual Contract

### StationInput.svelte

#### Input Field
- Background: `var(--color-bg-sunken)`
- Border: `1px solid var(--color-border-default)`
- Border on focus: `1px solid var(--color-border-focus)`
- Border radius: `0.75rem` (`rounded-xl`)
- Padding: `12px horizontal`, `10px vertical` (`px-3 py-2.5`)
- Font: `text-sm font-medium`, `color: var(--color-text-primary)`
- Placeholder: `color: var(--color-text-disabled)` (browser default for the token-styled input)

#### Dropdown
- Container: `glass-panel rounded-xl` (inherits glass morphism background, border, shadow, backdrop-filter)
- Position: `absolute z-20`, full width of parent (`left-0 right-0`), `mt-1` gap below input
- Max height: `192px` (`max-h-48`)
- Overflow: `overflow-y-auto` with hidden scrollbar (`no-scrollbar` class)
- Corner radius: `0.75rem` (`rounded-xl`)
- Overflow clipping: `overflow-hidden` on outer container to clip children to border-radius

#### Dropdown Option (idle)
- Full-width button: `w-full text-left px-3 py-2.5`
- Background: transparent
- Text: `text-sm`, `color: var(--color-text-primary)`
- English subtext: `text-xs ml-1`, `color: var(--color-text-tertiary)`
- Transition: `transition-colors`

#### Dropdown Option (active/highlighted via keyboard)
- Background: `var(--color-interactive-hover)`
- All other properties unchanged from idle state
- Applied via `aria-selected={true}` and conditional class: `class:bg-[var(--color-interactive-hover)]={i === activeIndex}`

#### Loading State
- Small spinner: 16px diameter circle with 2px stroke
- Stroke color: `var(--color-brand-text)`
- Position: right side of input field, vertically centered, `right-3`
- Animation: CSS `rotate` animation, 0.6s linear infinite
- Replaces placeholder text: no. Spinner overlays the trailing edge of the input.

#### Empty State (no matches found)
- Shown when: query length >= 1, API returned, suggestions array is empty, not loading
- Container: same dropdown container (glass-panel)
- Content: single `<li>` with `px-3 py-2.5 text-sm`, `color: var(--color-text-tertiary)`
- Copy: see Copywriting Contract below

## Interaction Contract

### Debounce
- Timer: 250ms (midpoint of 200-300ms range from D-03)
- Resets on every non-composition input event
- Skipped entirely while `isComposing === true`

### IME Composition
- `compositionstart`: set `isComposing = true`, suppress all fetch scheduling
- `compositionend`: set `isComposing = false`, immediately call `scheduleFetch(value)` with 250ms debounce
- Handles both Chrome (input before compositionend) and Firefox (input after compositionend) event orderings

### Keyboard Navigation (WAI-ARIA Combobox)
| Key | Dropdown closed | Dropdown open |
|-----|-----------------|---------------|
| ArrowDown | Open dropdown, highlight first option | Move highlight down (wrap to first) |
| ArrowUp | No action | Move highlight up (wrap to last) |
| Enter | Default form behavior | Select highlighted option, close dropdown |
| Escape | No action | Close dropdown, reset highlight to -1 |
| Tab | Default focus behavior | Close dropdown (via blur handler) |

- Highlight tracked via `activeIndex` state (-1 = no highlight)
- `aria-activedescendant` on input points to active option ID
- Active option auto-scrolls into view via `scrollIntoView({ block: 'nearest' })`
- `activeIndex` resets to -1 when suggestions array changes

### Focus/Blur
- On focus: if cached suggestions exist and input value matches last query, reshow dropdown immediately (D-06)
- On blur: 200ms delay before hiding dropdown (allows `onmousedown` on options to fire first)

### Mouse/Touch
- `onmousedown` (not onclick) on option buttons to fire before blur hides the dropdown
- Selection commits the Korean name to the input value

### Provider Change
- When `provider` prop changes, clear cached suggestions and close dropdown
- Implemented via `$effect` watching `provider`

### Stale Response Guard
- Each fetch records the query that triggered it
- On response, compare against current `value` -- discard if stale

## ARIA Contract

| Element | Attribute | Value |
|---------|-----------|-------|
| Input | `role` | `combobox` |
| Input | `aria-expanded` | `{expanded}` |
| Input | `aria-controls` | `{name}-station-listbox` |
| Input | `aria-activedescendant` | `{name}-station-option-{activeIndex}` or `undefined` |
| Input | `aria-autocomplete` | `list` |
| Dropdown `<ul>` | `id` | `{name}-station-listbox` |
| Dropdown `<ul>` | `role` | `listbox` |
| Option `<li>` | `id` | `{name}-station-option-{i}` |
| Option `<li>` | `role` | `option` |
| Option `<li>` | `aria-selected` | `{i === activeIndex}` |

`name` prop values: `dep` for departure, `arr` for arrival. This ensures unique IDs when two instances exist on the same page.

## Copywriting Contract

All user-facing strings go through `t()` from `$lib/i18n`. New i18n keys required for this phase:

| Element | i18n Key | Korean (ko) | English (en) | Japanese (ja) |
|---------|----------|-------------|--------------|---------------|
| Empty state (no matches) | `search.no_station_match` | 일치하는 역이 없습니다 | No matching stations | 一致する駅がありません |
| Loading indicator (aria-label) | `search.loading_stations` | 역 검색 중 | Searching stations | 駅を検索中 |

**Existing keys reused (no changes needed):**

| Element | i18n Key | Korean |
|---------|----------|--------|
| Departure label | `search.from` | 출발 |
| Arrival label | `search.to` | 도착 |
| Input placeholder | `search.select_station` | 역 선택 |
| Swap button aria-label | `search.swap_stations` | 역 바꾸기 |

**No primary CTA in this phase** -- the component is an input widget, not a page with a primary action button. The search button (`search.search_btn` / "검색") already exists and is unchanged.

**No destructive actions in this phase.**

**No error state copy needed** -- API errors silently result in an empty suggestions list. The existing `error.load_failed` key is not surfaced because failed station suggestions are non-critical (user can type the full station name manually).

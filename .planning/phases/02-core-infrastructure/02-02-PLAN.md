# Phase 2 / Plan 02

## Objective

Expose a stable typed app-domain surface in `bominal-app` by reusing the canonical backend/domain DTOs and validating serde round-trips.

## Implementation

1. Add a `crates/bominal-app/src/types.rs` module that re-exports or wraps the shared types the app consumes from `bominal-domain`.
2. Cover the typed surface expected by the current frontend: provider/task enums, auth payloads, DTOs, reservations, ticket info, Evervault/card types, and SSE task-event payloads.
3. Add focused serde round-trip tests for the app-facing type surface so the phase proves its "shared types compile and round-trip through serde" success criterion.
4. Export the typed module from `crates/bominal-app/src/lib.rs`.

## Verification

- `cargo test -p bominal-app --lib`
- `cargo check -p bominal-app --features ssr`
- `cargo check -p bominal-app --target wasm32-unknown-unknown --features hydrate`

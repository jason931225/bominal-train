# Phase 2 / Plan 03

## Objective

Create the typed `/api/` proxy layer and the baseline auth/theme/SSE state surfaces that later shell and page phases can consume.

## Implementation

1. Build a typed API module in `crates/bominal-app/src/api.rs` that mirrors the current `/api/...` contracts from the Svelte app and uses the Phase 2 typed surface.
2. Keep the architecture aligned with the roadmap decision to proxy existing `/api/` endpoints rather than reviving the donor crate's direct service-layer server functions.
3. Add a `crates/bominal-app/src/state.rs` module with auth, theme, and SSE stores/contexts patterned after the current Svelte store semantics.
4. Make the state providers accessible from `bominal-app` so Phase 3 can wire them into the root shell without redoing the infrastructure work.

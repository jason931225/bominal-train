# Phase 5 / Plan 02

## Objective

Replace the search stub with a functional train-search workflow including station autocomplete, date/time selection, search results, and task creation.

## Implementation

1. Build an app-local search page around the existing typed API surface instead of copying the donor modal/component stack.
2. Support provider switching, station suggestion inputs, native date/time controls, passenger and seat-preference state, and ordered target-train selection.
3. Submit a `CreateTaskInput` through `crate::api::create_task` and route the user into the live task surfaces on success.

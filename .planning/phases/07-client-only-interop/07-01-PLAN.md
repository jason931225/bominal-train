# Phase 7 / Plan 01

## Objective

Normalize the passkey ceremony boundary so login and registration run through the app-local Rust/browser interop layer, including conditional passkey login on the email form.

## Implementation

1. Move passkey browser calls behind `crates/bominal-app/src/browser.rs` so the WASM client does not call raw globals directly.
2. Port the donor conditional-passkey login support into `crates/bominal-app/src/api/passkey.rs`.
3. Update the auth UI to preserve manual passkey login while restoring email-form conditional mediation where supported.

- `cargo check -p bominal-app --target wasm32-unknown-unknown --features hydrate`

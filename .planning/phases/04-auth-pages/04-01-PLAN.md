# Phase 4 / Plan 01

## Objective

Replace the Phase 3 auth stubs for landing/login/signup/forgot with real page modules wired to the typed auth API surface and shared auth state.

## Implementation

1. Create or refine a reusable auth-shell helper in `pages/auth/mod.rs` so the landing, login, signup, and forgot routes share structure and styling.
2. Replace placeholder submit handlers in the existing app-local auth pages with calls into `crate::api::{login, register, forgot_password}` and update `AuthState` on success where appropriate.
3. Repoint the Phase 3 shell routes from `shell_pages.rs` stubs to the real page modules without disturbing the public/protected route contract.

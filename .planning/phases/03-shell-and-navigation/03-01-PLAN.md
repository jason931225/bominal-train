# Phase 3 / Plan 01

## Objective

Replace the Phase 1 placeholder with a real Leptos router, an auth-aware shell layout, and stub page components for every required route.

## Implementation

1. Add the missing app modules needed for a router-first shell: a lightweight browser helper for DOM attribute syncing and a page module containing reusable route stubs plus redirect components.
2. Update `state.rs` so theme defaults can be seeded from cookies and the shell can reflect the Phase 2 state surface immediately.
3. Rewrite `lib.rs` to install the router, bootstrap auth state via `get_me()`, split public vs protected layout rendering, and define all route paths required by the roadmap.

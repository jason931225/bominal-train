# Incident Management Standard

## Workflow

1. Triage scope and user impact.
2. Mitigate with safest reversible action.
3. Verify health and critical journeys.
4. Record root cause and preventive actions.

## Severity Guide

- SEV0: security/CDE boundary incident
- SEV1: major outage
- SEV2: partial outage/degradation
- SEV3: low-severity operational issue

## Break-Glass

Break-glass requires explicit approval, bounded window, exact commands, and rollback plan.
